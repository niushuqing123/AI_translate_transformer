# 机器翻译课程实验报告模板（可直接改写）

## 1. 实验目的

本实验旨在基于 Transformer 架构实现一个端到端机器翻译系统，并在小规模平行语料上完成从数据预处理、模型训练、结果评估到翻译演示的完整流程。实验重点不在于追求最优指标，而在于验证 Transformer 在机器翻译任务中的基本有效性，并分析不同模型设计或训练策略对翻译性能的影响。

## 2. 数据集说明

本实验选用 Multi30k 英语-德语平行语料。该数据集规模较小、文本干净、句子较短，适合作为课程作业中的轻量级实验数据集。

- 训练集：约 29000 句对
- 验证集：约 1014 句对
- 测试集：约 1000 句对

## 3. 数据预处理方法

本实验采用以下预处理流程：

1. 对原始句子进行统一小写化；
2. 使用正则规则进行基础分词；
3. 在训练集上分别构建源语言和目标语言词表；
4. 加入 `<pad>、<bos>、<eos>、<unk>` 等特殊符号；
5. 将句子转为索引序列，用于后续模型训练。

## 4. 模型设计

本实验实现的是一个轻量级 Encoder-Decoder Transformer，包含以下组件：

- 源语言嵌入层
- 目标语言嵌入层
- 位置编码
- 多层 Transformer Encoder
- 多层 Transformer Decoder
- 线性输出投影层

为了适配低显存环境，本实验采用了缩小后的模型参数设置，例如：

- `d_model = 128`
- `nhead = 4`
- `encoder layers = 2`
- `decoder layers = 2`
- `dim_feedforward = 256`

## 5. 训练设置

- 优化器：AdamW
- 学习率策略：Warmup + Inverse Square Root Decay
- 损失函数：Cross Entropy Loss
- 梯度裁剪：1.0
- 批大小：8
- 梯度累积：4
- 最大序列长度：40
- 训练轮数：根据配置文件自动设定

## 6. 评价指标

本实验使用以下指标评价模型性能：

- BLEU：衡量模型输出与参考翻译之间的 n-gram 匹配程度
- chrF：从字符层面评价翻译质量
- Token Accuracy：词级别命中率
- Exact Match：整句完全匹配率

## 11. 结论

本实验完成了一个基于 Transformer 的端到端机器翻译系统，并验证了该模型在小规模平行语料上的基本翻译能力。实验结果表明，在控制模型规模和显存开销的前提下，Transformer 仍然能够学习到一定的翻译映射关系。同时，对照实验也说明位置编码和标签平滑等设计会对模型最终效果产生影响。

## 12. 不足与改进方向

本实验仍存在一些不足，例如：

- 数据集规模较小，模型上限有限；
- 使用的是轻量级词表方案，未使用子词建模；
- 推理阶段采用贪心解码，可能影响最终 BLEU；
- 未做更复杂的调参和结构优化。

未来可进一步考虑：

- 引入 BPE / SentencePiece
- 使用 beam search
- 更换更大的平行语料
- 可视化注意力权重
